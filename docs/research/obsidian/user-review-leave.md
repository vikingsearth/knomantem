# Obsidian: Why Users Leave

Based on Product Hunt, Capterra, Software Advice, Reddit, blogs, and forums (2024-2026).

## 1. Steep Learning Curve
- "Not an app you master in an afternoon"
- Markdown unfamiliarity barriers non-technical users
- Understanding vaults, links, frontmatter, plugins requires time investment

## 2. Customization Rabbit Hole
- "I lost the forest for the trees and then blamed the forest"
- "Sometimes tempted to spend time perfecting the environment instead of taking notes"
- Infinite customizability becomes procrastination trap

## 3. Awkward Mobile Experience
- "UI is awkward on a smartphone — trying to replicate desktop UI"
- Initial loading slow, editing less polished than desktop
- Competitors have better mobile experiences

## 4. No Real-Time Collaboration
- "No inherent support for collaboration — problematic in team settings"
- Shared vaults via sync services are fragile
- Dealbreaker for team knowledge management

## 5. Sync Cost or Complexity
- Obsidian Sync $8/month; free alternatives can cause conflicts
- Git-based sync has its own learning curve

## 6. No Handwriting Support
- "For someone who journals by hand, this is non-negotiable"
- No stylus input, handwriting recognition, or OCR

## 7. PDF & Attachment Pain
- "PDF workflows still feel awkward"
- "No good solution to annotate, only link, to specific parts of a PDF"
- Attachment management is a chore

## 8. Plugin Overwhelm
- "The numerous plugins can get out of hand"
- Quality varies, some abandoned, some break with updates
- Plugin conflicts cause unexpected behavior

## 9. Default Appearance
- "Not a beautiful and super easy note-taking app"
- "A divisive app that some people love, while others delete it after five minutes"

## 10. "Blank Canvas" Problem
- No guidance on vault organization
- Analysis paralysis: Zettelkasten? PARA? Folders? Tags?
- High setup cost before seeing value

## Key Churn Insight
Churn is unique: no lock-in means users can leave easily, but most who reach proficiency stay. Critical window is first 2-4 weeks — if no working system established, likely to abandon. Learning curve is primary churn driver, not dissatisfaction with the tool itself.
