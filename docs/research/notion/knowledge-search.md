# Notion: Knowledge Search Capabilities

## Overview
Notion's search has evolved from a basic feature (historically its weakest point) to an AI-powered knowledge retrieval system. The Quick Find (Cmd/Ctrl+K) search bar is the primary entry point, and Notion AI Q&A transforms search from "find documents" to "get answers."

## Basic Search (Quick Find)
- Cmd/Ctrl+K for instant search across workspace
- Searches page titles, content, and database entries
- Recent pages list for quick re-access
- Filters by: creator, date, location (teamspace/workspace)
- Results show page title, snippet, and location
- Verified wiki pages get priority in results (blue check mark)

## AI-Powered Search (Business/Enterprise)
- "Search all sources with AI" option beneath search bar
- Searches workspace + connected apps + web
- Natural language queries: "What were the main challenges in last quarter's marketing campaign?"
- Cross-database search: finds related information across multiple databases simultaneously
- Contextual results: prioritizes based on current project and recent activity
- Returns answers (not just links) with citations

## AI Q&A
- Built-in chat interface for workspace knowledge retrieval
- Type a question, Q&A searches relevant pages/databases, returns synthesized answer
- Respects sharing and permission models
- Draws on AI Connectors (Google Drive, Slack, GitHub, Jira, Salesforce)
- Answers include links to sources for verification
- PDFs uploaded to Notion are now searchable

## Database Search & Filtering
- Filter database views by any property
- AND/OR filter groups for complex queries
- Sort by any property
- Linked databases surface filtered views across pages
- AI autofill properties make content more searchable via metadata

## AI Connectors (Cross-Platform Search)
- Read-only connections to external tools
- Supported: Slack, Google Drive, Microsoft Teams, Jira, Salesforce, and more
- Centralizes knowledge retrieval across tools
- "For searching, not acting" — read-only limitation

## Search Improvements in 2025
- PDF and image analysis — PDFs now searchable
- Multi-model AI (GPT-5, Claude) for better answer quality
- Notion 3.0 Agents can perform research across workspace
- Salesforce integration for CRM data search
- Image generation from search/prompt context

## Known Limitations

### Content Gaps
- @mentions of pages and people NOT included in search results
- Comments and discussions NOT included in search results
- Property values (select, multi-select tags) NOT included in search results
- These gaps can make search feel incomplete

### AI Limitations
- **36-72 hour sync delay** for AI Connectors — "big obstacle for teams that need things done now"
- **No real-time external data**: Can't query live databases or check shipping statuses
- **Short memory**: Struggles with complex databases and cross-question context
- **Closed-loop**: Only works with Notion and connected (read-only) data

### Pricing Barrier
- AI search features only on Business ($20/month) and Enterprise plans
- Free/Plus users get basic keyword search only
- Plus users get 20 lifetime AI responses (essentially a trial)
- "All-or-nothing" pricing — must pay for every workspace user

### Historical Pain Points
- Search was historically Notion's weakest feature
- "Once you get past a certain volume, it can be hard to find what you need"
- Performance with very large workspaces can degrade
- No regex or advanced query syntax for power users
- No search analytics (what people search for, failed searches)

## The Trajectory
Notion is investing heavily in search via AI — the direction is clearly "ask a question, get an answer" rather than "type keywords, get a list." This is a strong strategic bet, but the pricing barrier and sync delays limit its current impact for many teams.
