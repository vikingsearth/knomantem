# Flutter Navigation with go_router — Best Practices (2024/2025)

## Package Version

```yaml
dependencies:
  go_router: ^14.x.x   # Use latest stable; check pub.dev
```

`go_router` is Flutter's officially recommended routing package (maintained by the Flutter team). It implements the `Router` API, integrates with the browser History API on web, and supports deep links, URL parameters, shell routes, and redirects.

---

## 1. Basic Route Configuration

Define all routes in a single file (`core/router/app_router.dart`) to make navigation auditable.

```dart
// core/router/app_router.dart
import 'package:go_router/go_router.dart';
import 'package:riverpod_annotation/riverpod_annotation.dart';

part 'app_router.g.dart';

@riverpod
GoRouter appRouter(AppRouterRef ref) {
  // Watch auth state to trigger redirects reactively
  final authState = ref.watch(authStateProvider);

  return GoRouter(
    initialLocation: '/home',
    debugLogDiagnostics: kDebugMode,
    routes: $appRoutes, // Generated by go_router_builder (optional)
    redirect: (context, state) => _globalRedirect(authState, state),
    errorBuilder: (context, state) => ErrorScreen(error: state.error),
  );
}
```

Plug into the app:
```dart
// main.dart
@override
Widget build(BuildContext context, WidgetRef ref) {
  final router = ref.watch(appRouterProvider);
  return MaterialApp.router(
    routerConfig: router,
    theme: AppTheme.light,
    // ...
  );
}
```

## 2. Route Definitions

### Flat Routes

```dart
GoRoute(
  path: '/articles',
  name: 'articleList',
  builder: (context, state) => const ArticleListScreen(),
),
GoRoute(
  path: '/articles/:id',
  name: 'articleDetail',
  builder: (context, state) {
    final id = state.pathParameters['id']!;
    return ArticleDetailScreen(articleId: id);
  },
),
```

### Query Parameters

```dart
GoRoute(
  path: '/search',
  builder: (context, state) {
    final query = state.uri.queryParameters['q'] ?? '';
    return SearchScreen(initialQuery: query);
  },
),
```

Navigate with query params:
```dart
context.go('/search?q=flutter+riverpod');
```

### Extra Data (Non-URL State)

Pass objects that do not need to survive a browser refresh through `extra`:
```dart
context.go('/articles/preview', extra: draftArticle);

// Receive in builder:
GoRoute(
  path: '/articles/preview',
  builder: (context, state) {
    final draft = state.extra as Article;
    return ArticlePreviewScreen(article: draft);
  },
),
```

Use `extra` for in-memory navigation only. Deep links and browser refresh will not have `extra` data — handle the null case.

### Using Named Routes (Recommended)

Navigate by name to avoid hard-coded path strings scattered through the codebase:

```dart
context.goNamed('articleDetail', pathParameters: {'id': article.id});
context.pushNamed('settings');
```

## 3. ShellRoute — Persistent Navigation Shell

`ShellRoute` wraps child routes inside a persistent shell widget (e.g., a `Scaffold` with `NavigationRail` or `BottomNavigationBar`). The shell is not rebuilt when navigating between child routes.

```dart
ShellRoute(
  builder: (context, state, child) => AppShell(child: child),
  routes: [
    GoRoute(
      path: '/home',
      name: 'home',
      builder: (context, state) => const HomeScreen(),
    ),
    GoRoute(
      path: '/knowledge-base',
      name: 'knowledgeBase',
      builder: (context, state) => const KnowledgeBaseScreen(),
      routes: [
        GoRoute(
          path: ':id',
          name: 'articleDetail',
          builder: (context, state) =>
              ArticleDetailScreen(articleId: state.pathParameters['id']!),
        ),
      ],
    ),
    GoRoute(
      path: '/settings',
      name: 'settings',
      builder: (context, state) => const SettingsScreen(),
    ),
  ],
),
```

The `AppShell` widget receives the current route's widget as `child`:

```dart
class AppShell extends StatelessWidget {
  const AppShell({super.key, required this.child});
  final Widget child;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Row(
        children: [
          NavigationRail(
            selectedIndex: _selectedIndex(context),
            onDestinationSelected: (index) => _navigate(context, index),
            destinations: const [
              NavigationRailDestination(icon: Icon(Icons.home), label: Text('Home')),
              NavigationRailDestination(icon: Icon(Icons.book), label: Text('Knowledge Base')),
              NavigationRailDestination(icon: Icon(Icons.settings), label: Text('Settings')),
            ],
          ),
          Expanded(child: child),
        ],
      ),
    );
  }

  int _selectedIndex(BuildContext context) {
    final location = GoRouterState.of(context).matchedLocation;
    if (location.startsWith('/knowledge-base')) return 1;
    if (location.startsWith('/settings')) return 2;
    return 0;
  }

  void _navigate(BuildContext context, int index) {
    switch (index) {
      case 0: context.go('/home');
      case 1: context.go('/knowledge-base');
      case 2: context.go('/settings');
    }
  }
}
```

### `StatefulShellRoute` — Preserved State Per Branch

`StatefulShellRoute` (go_router 7+) preserves the navigation state of each branch when switching tabs. This means scroll positions and sub-navigation within each branch are preserved:

```dart
StatefulShellRoute.indexedStack(
  builder: (context, state, navigationShell) {
    return ScaffoldWithNavBar(navigationShell: navigationShell);
  },
  branches: [
    StatefulShellBranch(
      routes: [
        GoRoute(path: '/home', builder: (_, __) => const HomeScreen()),
      ],
    ),
    StatefulShellBranch(
      routes: [
        GoRoute(
          path: '/knowledge-base',
          builder: (_, __) => const KnowledgeBaseScreen(),
          routes: [
            GoRoute(
              path: ':id',
              builder: (context, state) =>
                  ArticleDetailScreen(articleId: state.pathParameters['id']!),
            ),
          ],
        ),
      ],
    ),
    StatefulShellBranch(
      routes: [
        GoRoute(path: '/settings', builder: (_, __) => const SettingsScreen()),
      ],
    ),
  ],
),
```

Use `StatefulShellRoute` when users need to return to their position in one tab after visiting another. Use plain `ShellRoute` when the shell is purely visual and branches don't maintain state.

## 4. Redirects and Auth Guard

### Global Redirect

The `redirect` callback on `GoRouter` runs before every navigation. Return `null` to allow the navigation, or return a path string to redirect.

```dart
String? _globalRedirect(AuthState authState, GoRouterState state) {
  final isOnAuthRoute = state.matchedLocation.startsWith('/auth');

  return switch (authState) {
    AuthStateAuthenticated() => isOnAuthRoute ? '/home' : null,
    AuthStateUnauthenticated() => isOnAuthRoute ? null : '/auth/login',
    AuthStateLoading() => '/splash',
  };
}
```

### Reactive Redirect via `refreshListenable`

When auth state changes (e.g., token expires), the router should re-evaluate all redirects automatically. Pass a `Listenable` to `refreshListenable`:

```dart
GoRouter(
  redirect: (context, state) => _globalRedirect(authState, state),
  refreshListenable: GoRouterRefreshStream(authStateStream),
);

// Helper to convert Stream to Listenable (or use a ChangeNotifier)
class GoRouterRefreshStream extends ChangeNotifier {
  GoRouterRefreshStream(Stream<dynamic> stream) {
    _subscription = stream.asBroadcastStream().listen((_) => notifyListeners());
  }
  late final StreamSubscription<dynamic> _subscription;

  @override
  void dispose() {
    _subscription.cancel();
    super.dispose();
  }
}
```

With Riverpod, a cleaner pattern is to use a `ValueNotifier` wrapped around the auth provider:

```dart
@riverpod
GoRouter appRouter(AppRouterRef ref) {
  final authState = ref.watch(authStateProvider);

  // Create a ValueNotifier that rebuilds the router when auth changes
  final notifier = ValueNotifier<AuthState>(authState);
  ref.listen(authStateProvider, (_, next) => notifier.value = next);
  ref.onDispose(notifier.dispose);

  return GoRouter(
    refreshListenable: notifier,
    redirect: (context, state) => _globalRedirect(notifier.value, state),
    routes: [...],
  );
}
```

### Per-Route Redirect

```dart
GoRoute(
  path: '/admin',
  redirect: (context, state) {
    final user = GoRouterState.of(context).extra as User?;
    return user?.isAdmin == true ? null : '/home';
  },
  builder: (_, __) => const AdminScreen(),
),
```

## 5. Deep Links

### Path URL Strategy (Required for Clean URLs)

```dart
// main.dart — call before runApp
import 'package:flutter_web_plugins/url_strategy.dart';

void main() {
  usePathUrlStrategy(); // Switches from /#/path to /path
  runApp(const MyApp());
}
```

Configure your web server to rewrite all paths to `index.html`. For Firebase Hosting:
```json
{ "hosting": { "rewrites": [{ "source": "**", "destination": "/index.html" }] } }
```

### iOS Deep Link Setup

In `ios/Runner/Info.plist`:
```xml
<key>FlutterDeepLinkingEnabled</key>
<true/>
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>myapp</string>
    </array>
  </dict>
</array>
```

For Universal Links (HTTPS), add your domain to `Associated Domains` entitlement and host an `apple-app-site-association` file.

### Android Deep Link Setup

In `android/app/src/main/AndroidManifest.xml`:
```xml
<intent-filter android:autoVerify="true">
  <action android:name="android.intent.action.VIEW" />
  <category android:name="android.intent.category.DEFAULT" />
  <category android:name="android.intent.category.BROWSABLE" />
  <data android:scheme="https" android:host="app.example.com" />
</intent-filter>
```

go_router handles incoming deep links automatically once the OS routes the intent/URL to the app.

## 6. Navigation Patterns

### `go` vs `push` vs `replace`

| Method | Behaviour | Use When |
|---|---|---|
| `context.go('/path')` | Replace entire navigation stack | Tabs, main navigation |
| `context.push('/path')` | Push on top of current stack | Drill-down, detail screens |
| `context.replace('/path')` | Replace current route in stack | Wizard/onboarding steps |
| `context.pop()` | Go back one level | Modal dismiss, back button |
| `context.canPop()` | Check if pop is possible | Conditional back button display |

### Returning a Result from a Pushed Route

```dart
// Caller
final result = await context.push<String>('/article/edit', extra: article);
if (result != null) {
  showSnackBar('Saved: $result');
}

// Called screen
context.pop('Article saved successfully');
```

### Programmatic Navigation from Outside Widget Tree

When navigating from a Riverpod notifier (outside widget tree), hold a reference to the router:

```dart
@Riverpod(keepAlive: true)
GoRouter appRouter(AppRouterRef ref) { ... }

// In a notifier:
ref.read(appRouterProvider).go('/home');
```

## 7. Common Mistakes

- Defining routes with overlapping path patterns — go_router matches top-to-bottom; order matters.
- Using `context.go` for drill-down navigation — this clears the stack. Use `context.push` for screens you want the back button to return from.
- Hardcoding paths as strings in multiple places — use named routes or route constants.
- Forgetting to configure the web server for path URL strategy — results in 404 on direct URL access or refresh.
- Putting `GoRouter` creation inside `build()` — creates a new router on every rebuild. Always create it in a Riverpod provider or `initState`.
